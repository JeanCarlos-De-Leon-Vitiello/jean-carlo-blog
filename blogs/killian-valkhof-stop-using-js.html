Killian Valkhof
Stop using JS.. In the future

Wat hij vertelde

Tijdens de Weekly Nerd vertelde Killian Valkhof dat developers nog te veel JavaScript gebruiken terwijl HTML en CSS tegenwoordig veel mogelijkheden hebben. Zijn belangrijkste boodschap was:

kies altijd de minst krachtige oplossing die nodig is

Browsers kunnen veel sneller omgaan met HTML en CSS dan met JavaScript. Zodra je JavaScript gebruikt, moet de browser extra werk doen om alles uit te voeren en te renderen. Hij vertelde ook dat oude websites vaak nog steeds blijven werken in moderne browsers. Dat komt doordat browsers achterwaarts compatibel zijn gemaakt. Oude code blijft dus ondersteund zodat websites niet ineens kapot gaan wanneer een browser update krijgt. Als laatst liet hij voorbeelden zien van functionaliteiten die vroeger alleen met JavaScript mogelijk waren, maar nu gewoon met CSS en HTML gemaakt kunnen worden.

Belangrijke inzichten

  • Custom switch zonder JavaScript

    Met appearance: none kan je de standaard styling van een input verwijderen. Hierdoor verandert een checkbox eigenlijk in een leeg element dat je volledig zelf kunt stylen. Door vervolgens pseudo-elementen zoals ::before te gebruiken kan je bijvoorbeeld een bolletje maken binnen de switch. Met :checked kan je de staat aanpassen zodra iemand erop klikt. Hierdoor kan je een interactieve switch maken zonder één regel JavaScript te gebruiken. Killian vertelde hierbij ook dat veel developers nog denken dat forms moeilijk te stijlen zijn. Volgens hem was dat vroeger misschien waar, maar tegenwoordig bieden browsers veel betere mogelijkheden om formulier-elementen aan te passen.

  • Smooth scrolling zonder JavaScript

    Voor smooth scrolling gebruikte Killian alleen CSS: html {scroll-behavior: smooth;} Hierdoor scrollt de browser automatisch met een animatie naar een sectie van de pagina. Hiervoor hoef je dus geen JavaScript meer te gebruiken.

  • Reduce motion

    Als iemand “reduce motion” heeft aangezet in zijn browser of besturingssysteem, worden animaties uitgeschakeld. Dit is belangrijk voor toegankelijkheid en gebruiksvriendelijkheid. @media (prefers-reduced-motion: no-preference) {html {scroll-behavior: smooth;}}

  • Scroll-margin

    Met scroll-margin kan je voorkomen dat content onder een sticky header verdwijnt wanneer je naar een ankerlink scrollt.

  • Accordions met details en summary

    Killian liet zien dat je tegenwoordig accordions kunt maken met alleen HTML, zonder JavaScript te gebruiken. Met de elementen details en summary kan je eenvoudig een uitklapbaar onderdeel maken waarbij alleen de titel zichtbaar is totdat iemand erop klikt. Wanneer het onderdeel geopend wordt, voegt de browser automatisch een open status toe. Daarnaast kan je het standaard pijltje aanpassen met CSS. Ook liet hij zien dat je met het name attribuut meerdere accordions aan elkaar kunt koppelen, zodat er maar één tegelijk open kan staan. Open je een nieuwe accordion, dan sluit de vorige automatisch.

  • Dialog

    Tot slot vertelde hij over het dialog element. Dit is eigenlijk een ingebouwde popup van HTML. Het element is standaard verborgen totdat je het opent. Met :backdrop kan je de achtergrond achter de popup stijlen, bijvoorbeeld donker maken. Ook @starting-style kan gebruikt worden om dialogs te animeren.

Wat neem ik mee

Na deze Weekly Nerd wil ik bewuster omgaan met het gebruik van JavaScript. Ik wil eerst kijken of iets met HTML en CSS opgelost kan worden voordat ik direct JavaScript toevoeg. Daarnaast wil ik meer aandacht besteden aan accessibility, bijvoorbeeld door betere focus states toe te voegen en rekening te houden met gebruikers die minder animaties willen zien.
